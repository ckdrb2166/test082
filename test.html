it init git status git add . -> 폴더내 변경사항 업데이트 git add 폴더명 -> 업데이트할 폴더명 git commit -m 'practice' git remote add origin 원격주소 git push origin 해당 branch 터미널로 해당 프로젝트 폴더로 이동 git init git add . git commit -m "message" git remote add origin "github.com/your_repo.git" git push -u origin master 1. git init - 깃으로 관리할 프로젝트 지정(처음 한번만 해주면 된다.) + 현재 디렉터리를 새로운 깃 저장소로 초기화 한다는 것이다. - 참고로 git init을 하면 기본적으로 master 브랜치가 자동으로 생긴다 ​ 2. git add . - 깃에는 staging area라는 공간이 있는데, 먼저 여기에 저장할 파일들을 선택해서 넣어두고, 나중에 staging area에 있는 파일들을 통째로 저장한다. 예) git add file1 file2 file3 - 만약 파일이 많아서 일일이 써주기 힘든 경우에는 git add.(현재 디렉터리에 있는 모든 파일을 staging area로 보냄) 3. git commit -m "first commit" - git add로 staging area에 올린 파일들을 저장해주는 명령어 - 여기서 "first commit"은 일종의 메시지, 메모 같은 것이다. 나중에 commit한 내용을 찾기가 더 수월해진다. 4. git remote add origin "https://github.com/hoony0125/hoony0125.github.io.git" - 여기서 레포지터리 주소를 orgin이라는 이름으로 추가해준다는 뜻이며, 그러니 orgin이라는 이름으로 이 주소의 레포지터리의 코드를 업로드 또는 다운로드가 가능하다는 것이다. 5. git push -u origin master - git push는 레포지터리에 작업한 내용을 올리는 명령어이다 - 링크의 이름이 orgin인 레포지터리에 master라는 이름의 브랜치를 올린다는 것이다 6. github에 올라간 파일들을 확인한다. - settings에서 gihub pages를 보면 된다.
